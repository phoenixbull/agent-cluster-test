## 📋 需求概述

### 项目背景

随着短剧市场的爆发式增长，内容创作需求巨大。利用 AI 技术自动化短剧创作流程，可以大幅提升创作效率，降低制作成本。

### 项目目标

搭建一个 AI 生成短剧的智能体工作流系统，实现从创意到剧本的自动化生成。

### 核心要求

1. ✅ **每个阶段都单独 Agent 负责** - 专业化分工，提高质量
2. ✅ **选择更适合大模型调用的技术栈** - Python + FastAPI 后端，React + TypeScript 前端
3. ✅ **先搭建整体工作流** - 框架优先，Agent 实现后续完善

## 📐 技术架构

### 后端技术栈

- **语言**: Python 3.11+
- **框架**: FastAPI
- **数据库**: PostgreSQL + Redis
- **工作流引擎**: Temporal.io
- **消息队列**: RabbitMQ
- **大模型**: Qwen-Max / GPT-4 / Claude

### 前端技术栈

- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI 库**: shadcn/ui + TailwindCSS
- **状态管理**: Zustand
- **图表**: Recharts

### 部署架构

- **容器**: Docker + Docker Compose
- **编排**: Kubernetes（可选）
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack
