# 📋 产品需求文档 (PRD)

**项目名称**: 个人博客系统  
**版本**: 1.0  
**创建时间**: 2026-03-09  
**创建者**: Product Manager Agent

---

## 1. 项目概述

### 1.1 项目背景
用户需要一个个人博客系统来记录和分享知识，要求功能完善、界面简洁、易于使用。

### 1.2 产品目标
- ✅ 提供流畅的博客写作体验
- ✅ 支持文章分类管理
- ✅ 实现用户互动（评论）
- ✅ 支持 Markdown 编辑
- ✅ 现代简洁的界面设计

### 1.3 成功指标
- 文章发布成功率 > 99%
- 页面加载时间 < 2 秒
- 用户满意度 > 4.5/5
- 移动端适配率 100%

---

## 2. 目标用户

### 2.1 用户画像

**主要用户**: 技术博主/知识分享者
- 年龄：25-40 岁
- 职业：开发者、产品经理、设计师
- 需求：记录技术文章、分享经验
- 痛点：现有平台广告多、不自由

**次要用户**: 读者
- 年龄：20-45 岁
- 需求：阅读优质内容、互动交流
- 痛点：评论体验差、找不到好内容

### 2.2 使用场景
1. **写作场景**: 安静环境，专注写作
2. **阅读场景**: 碎片时间，移动端为主
3. **互动场景**: 看到好文章，想评论交流

### 3.2 功能详细说明

#### F001 - 用户注册登录

**用户故事**:
```
As a 访客
I want 注册账号并登录
So that 可以发布文章和评论
```

**验收标准**:
```
Given 用户访问注册页面
When 填写邮箱、密码并点击注册
Then 注册成功并自动登录

Given 用户已注册
When 输入正确的邮箱密码登录
Then 登录成功并跳转到首页

Given 用户输入错误密码
When 点击登录
Then 显示错误提示"密码错误"
```

**功能点**:
- 邮箱格式验证
- 密码强度检查（最少 6 位）
- 邮箱唯一性验证
- JWT Token 认证
- 记住登录状态（7 天）

---

#### F002 - 文章管理

**用户故事**:
```
As a 博主
I want 创建、编辑、删除文章
So that 可以管理我的内容
```

**验收标准**:
```
Given 用户已登录
When 点击"写文章"按钮
Then 进入文章编辑页面

Given 用户在编辑页面
When 填写标题、内容并发布
Then 文章发布成功并显示在列表中

Given 用户有一篇文章
When 点击删除并确认
Then 文章被删除
```

**功能点**:
- 草稿自动保存（每 30 秒）
- 文章状态：草稿/已发布
- 发布时间可设置（定时发布）
- 支持文章置顶

---

#### F003 - Markdown 编辑器

**用户故事**:
```
As a 博主
I want 使用 Markdown 编辑器写作
So that 可以高效排版和插入代码
```

**验收标准**:
```
Given 用户在编辑页面
When 输入 Markdown 语法
Then 实时预览渲染效果

Given 用户插入代码块
When 选择编程语言
Then 代码高亮显示

Given 用户上传图片
When 拖拽图片到编辑器
Then 图片上传并插入到光标位置
```

**功能点**:
- 实时预览（左右分栏）
- 代码高亮（支持 20+ 语言）
- 图片拖拽上传
- 目录自动生成
- 快捷键支持

---

#### F004 - 分类管理

**用户故事**:
```
As a 博主
I want 创建和管理文章分类
So that 可以组织我的文章
```

**验收标准**:
```
Given 用户已登录
When 创建新分类
Then 分类出现在列表中

Given 用户有一个分类
When 删除该分类
Then 分类下的文章移动到"未分类"
```

**功能点**:
- 分类层级（支持二级分类）
- 分类排序
- 分类文章计数
- 分类颜色标记

---

#### F005 - 评论系统

**用户故事**:
```
As a 读者
I want 对文章发表评论
So that 可以和作者互动
```

**验收标准**:
```
Given 用户已登录
When 在文章底部发表评论
Then 评论显示在评论列表中

Given 用户想回复评论
When 点击某条评论的"回复"
Then 可以回复该评论
```

**功能点**:
- 评论审核（可配置）
- 评论回复（嵌套 3 层）
- 评论点赞
- 评论举报
- 作者回复标识

## 4. 非功能需求

### 4.1 性能需求
- 首页加载时间 < 2 秒
- 文章列表加载 < 1.5 秒
- 文章发布响应 < 1 秒
- 支持并发用户数 > 1000

### 4.2 安全需求
- 密码加密存储（bcrypt）
- JWT Token 认证
- XSS 防护
- CSRF 防护
- SQL 注入防护

### 4.3 可用性需求
- 系统可用性 > 99%
- 数据备份（每日自动）
- 错误日志记录
- 异常自动恢复

### 4.4 兼容性需求
- 浏览器：Chrome 80+, Firefox 75+, Safari 13+
- 移动端：iOS 12+, Android 8+
- 屏幕尺寸：320px - 2560px

---

## 5. 数据需求

### 5.1 数据模型

**用户表 (users)**:
- id, email, password_hash, nickname, avatar, created_at

**文章表 (posts)**:
- id, user_id, title, content, status, view_count, created_at, updated_at

**分类表 (categories)**:
- id, user_id, name, slug, parent_id, sort_order

**评论表 (comments)**:
- id, post_id, user_id, content, parent_id, status, created_at

### 5.2 数据量预估
- 用户数：1 万
- 文章数：10 万
- 评论数：50 万
- 存储需求：50GB
