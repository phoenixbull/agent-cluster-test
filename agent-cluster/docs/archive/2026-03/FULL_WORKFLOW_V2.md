## 🎯 完整工作流程详解

### 阶段 0: 需求接收与项目识别

#### 输入方式
```bash
# 方式 1: 命令行
python3 orchestrator.py "[电商] 添加购物车功能"

# 方式 2: 钉钉机器人 (待实现 Webhook)
@Agent 集群助手 [电商] 添加购物车功能

# 方式 3: OpenClaw 会话
sessions_spawn --agentId orchestrator --task "[电商] 添加购物车功能"
```

#### 项目识别流程
```
输入："[电商] 添加购物车功能"
         ↓
┌─────────────────────────┐
│ 1. 检查前缀标记          │
│    [电商] → ecommerce   │
└─────────────────────────┘
         ↓ (无前缀时)
┌─────────────────────────┐
│ 2. 关键词匹配            │
│    购物车→电商 (2 个匹配)  │
│    文章→博客 (3 个匹配)   │
└─────────────────────────┘
         ↓ (无匹配时)
┌─────────────────────────┐
│ 3. 默认项目              │
│    default              │
└─────────────────────────┘
```

#### 输出示例
```
🔍 识别项目...
   📁 识别项目 (关键词): 电商项目 (匹配 2 个关键词)
   📁 项目：电商项目
   🗂️ 仓库：phoenixbull/ecommerce-platform
   📂 工作区：/home/admin/.openclaw/workspace/ecommerce
   ✅ GitHub 客户端已初始化
   📁 设置项目工作区：/home/admin/.openclaw/workspace/ecommerce
```
