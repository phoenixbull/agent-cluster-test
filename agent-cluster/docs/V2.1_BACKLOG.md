### 1. 大文件函数级修改

**问题**: 当前大文件 (>8000 字符) 只添加 TODO 注释，效果不佳

**现状**:
```python
def _apply_minimal_changes(self, code: str, feedback: Dict) -> str:
    # 简化实现：添加 TODO 注释
    fix_comments = "\n# 🔧 TODO: 需要修复以下问题:\n"
    for issue in feedback.get('critical_issues', []):
        fix_comments += f"# - {issue.get('description', '未描述')}\n"
    return code + fix_comments
```

**目标**:
```python
def _apply_function_level_changes(self, code: str, feedback: Dict) -> str:
    """
    函数级增量修改 (针对大文件)
    
    策略:
    1. 使用 AST 分析识别需要修改的函数
    2. 只提取相关函数 + 上下文
    3. 修改后合并回原文件
    """
    import ast
    tree = ast.parse(code)
    
    # 识别需要修改的函数
    functions_to_modify = self._identify_functions_to_modify(tree, feedback)
    
    # 对每个函数单独修改
    for func in functions_to_modify:
        func_code = ast.unparse(func)
        modified_func = self._modify_single_function(func_code, feedback)
        # 替换原函数
    
    return code
```

**实施成本**: 半天 (4 小时)

**触发条件** (满足任一即实施):
- [ ] 出现>3 次大文件修改需求
- [ ] 用户反馈 TODO 注释不够用
- [ ] MVP 运行数据显示大文件修改失败率>50%

**实施计划**:
```
Day 3 上午 (如决定实施):
- 实现 _identify_functions_to_modify() - 1 小时
- 实现 _modify_single_function() - 2 小时
- 测试验证 - 1 小时
```

def _execute_incremental_fix(self, coding_result: Dict, review_result: Dict) -> Dict:
