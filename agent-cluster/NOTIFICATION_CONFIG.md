## 📱 用户交互流程

### 部署确认流程

```
1. 代码审查通过
   ↓
2. 发送钉钉确认通知 (@所有人)
   ↓
3. 等待用户回复
   │
   ├─→ 回复"部署" → 执行部署 → 发送完成通知
   │
   ├─→ 回复"取消" → 取消部署 → 发送取消通知
   │
   └─→ 超时 30 分钟 → 自动取消 → 发送超时通知
```

### 失败处理流程

```
1. 任务执行失败
   ↓
2. 发送钉钉失败告警 (@所有人)
   ↓
3. 等待用户回复
   │
   ├─→ 回复"重试" → 重新执行任务
   │
   └─→ 回复"跳过" → 跳过任务继续
```
