
**处理流程**:
```
1. 发送确认通知 (@所有人)
   ↓
2. 等待用户回复
   │
   ├─→ 回复"部署 XXX" → ✅ 批准 → 执行部署
   │
   ├─→ 回复"取消 XXX" → ❌ 取消 → 发送取消通知
   │
   └─→ 超时 30 分钟 → ⏱️ 超时 → 自动取消
```

**特点**:
- ⚠️ 重要决策
- ✅ @所有人
- ✅ 需要确认
- ⏱️ 30 分钟超时

## ⚠️ 重要说明

### 部署确认机制

1. **生产环境** 必须经过钉钉确认才能部署
2. **测试环境** 可以配置为自动部署
3. 确认超时时间为 **30 分钟**
4. 超时后自动取消，需要重新触发

### 安全机制

1. 部署确认 ID 每次不同，防止误操作
2. 只有@所有人才能看到确认通知
3. 支持多人确认（第一个回复有效）
4. 所有操作有日志记录

### 最佳实践

1. **及时响应** - 看到确认通知尽快回复
2. **仔细检查** - 确认前检查变更内容
3. **超时处理** - 如无法及时确认，先取消再约时间
4. **记录归档** - 所有确认记录自动保存
