# 阶段 1: 需求分析

## 📋 项目概述

**项目名称**: Task Dashboard 任务管理系统  
**版本**: 2.3.0  
**技术栈**: FastAPI + React + TypeScript + JWT

## 🎯 核心需求

### 功能需求

1. **用户认证**
   - 用户注册（邮箱、用户名、密码）
   - 用户登录（JWT Token）
   - Token 刷新机制
   - 密码加密存储（bcrypt）

2. **任务管理**
   - 创建任务（标题、描述、优先级、截止日期）
   - 查看任务列表（分页、筛选）
   - 更新任务（状态、优先级等）
   - 删除任务
   - 任务统计（按状态、优先级）

3. **权限控制**
   - 用户只能管理自己的任务
   - JWT 认证保护所有 API

### 非功能需求

| 指标 | 目标 | 验收标准 |
|------|------|----------|
| 测试覆盖率 | ≥80% | pytest-cov 报告 |
| 代码审查评分 | ≥80/100 | CodeReview 工具 |
| API 响应时间 | P95 < 200ms | 性能测试 |
| 安全合规 | OWASP Top 10 | 安全扫描 |

## 👥 用户故事

### US-001: 用户注册
> 作为新用户，我希望能够注册账号，以便使用任务管理功能。

**验收标准**:
- [ ] 输入邮箱、用户名、密码
- [ ] 密码强度验证（8 位以上，包含大小写字母和数字）
- [ ] 邮箱格式验证
- [ ] 用户名唯一性检查
- [ ] 注册成功后可登录

### US-002: 用户登录
> 作为已注册用户，我希望能够登录系统，以便访问我的任务。

**验收标准**:
- [ ] 输入邮箱和密码
- [ ] 验证失败显示友好错误
- [ ] 登录成功返回 JWT Token
- [ ] Token 自动添加到后续请求

### US-003: 创建任务
> 作为已登录用户，我希望能够创建新任务，以便记录待办事项。

**验收标准**:
- [ ] 输入任务标题（必填）
- [ ] 可选填写描述、优先级、截止日期
- [ ] 创建后显示在任务列表
- [ ] 默认状态为"待处理"

### US-004: 管理任务
> 作为任务所有者，我希望能够更新和删除任务，以便保持任务列表准确。

**验收标准**:
- [ ] 可修改任务状态（待处理/进行中/已完成/已取消）
- [ ] 可修改优先级和截止日期
- [ ] 完成任务自动记录完成时间
- [ ] 可删除自己的任务

### US-005: 查看统计
> 作为用户，我希望能够查看任务统计，以便了解我的工作进度。

**验收标准**:
- [ ] 显示总任务数
- [ ] 按状态分类统计
- [ ] 按优先级分类统计
- [ ] 显示过期任务数
- [ ] 显示今日完成任务数

## 📊 数据模型

### User 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| email | String(255) | 邮箱（唯一） |
| username | String(100) | 用户名（唯一） |
| hashed_password | String(255) | 密码哈希 |
| is_active | Boolean | 是否激活 |
| created_at | DateTime | 创建时间 |

### Task 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| title | String(200) | 标题 |
| description | Text | 描述 |
| status | Enum | 状态 |
| priority | Enum | 优先级 |
| due_date | DateTime | 截止日期 |
| owner_id | Integer | 所有者外键 |
| created_at | DateTime | 创建时间 |
